## SECTION 1: CLINOSTAT GEOMETRY & DESIGN — UNIQUE PER TYPE

### ⚙️ 1D CLINOSTAT — Geometry & Design

**Physical Description:**
- Single rotation axis — horizontal, fixed
- One motor drives one cylindrical/rectangular outer frame
- Sample (payload) sits at geometric center of the rotating frame
- Frame is typically a flat disc or rectangular cage
- Rotation at 1–10 RPM (slow) or up to 60 RPM (fast clinostat)

**Geometric Parameters (all user-adjustable in UI):**
- Frame outer diameter: D_outer (default 200mm)
- Frame inner diameter: D_inner (default 80mm) — payload space
- Frame thickness/depth: t (default 20mm)
- Frame material: selectable (Aluminum 6061, Acrylic, PLA, PEEK)
- Payload chamber: cylindrical, centered, diameter selectable (default 60mm), height selectable (default 50mm)
- Motor mount: rear axial mount
- Bearing type: selectable (deep groove ball bearing, angular contact)

**CAD Representation:**
- Render as parametric 3D model in Three.js
- Show: outer ring, inner payload cylinder, motor shaft, bearing seats, mounting brackets
- Highlight center of rotation with crosshair axis
- Show gravity vector arrow (downward, red) and time-averaged gravity vector (blue, approaches zero with rotation)

**1D Physics:**
- Gravity vector in rotating frame:
  ```
  g_x(t) = g * sin(ω*t)
  g_y(t) = g * cos(ω*t)
  g_z(t) = 0  [along rotation axis]
  ```
- Time-averaged: g_avg = 0 (ideal), residual depends on RPM and sample offset from center
- Centrifugal acceleration: a_c = ω² * r (must be minimized by keeping sample at r ≈ 0)
- Sedimentation nullification condition: ω >> (2*g) / (π * v_s) where v_s is Stokes settling velocity

**Limitations to display in UI:**
- Only nullifies gravity in 2D plane — z-axis gravity persists along rotation axis
- Not suitable for elongated organisms or asymmetric samples
- Effective only for slow-settling biological samples

### ⚙️ 2D CLINOSTAT / RANDOM POSITIONING MACHINE (RPM) — Geometry & Design

**Physical Description:**
- Two independent rotation axes — inner and outer frames (gimbal configuration)
- Outer frame rotates about vertical or horizontal axis
- Inner frame rotates about an axis perpendicular to outer
- Two independent motors (one per axis)
- Sample sits at geometric center of inner frame
- Can operate in: Constant Speed mode, Random Walk mode, Optimized RPM mode

**Geometric Parameters (all user-adjustable):**
- Outer frame: rectangular or circular, outer width W_outer (default 350mm), height H_outer (default 350mm)
- Inner frame: circular ring, diameter D_inner_frame (default 200mm), depth 30mm
- Frame material: selectable
- Payload chamber: sphere or cylinder, diameter selectable (default 60mm)
- Axle length, diameter: adjustable
- Inter-frame clearance: minimum 10mm, validated by collision detection engine
- Motor mount: one on each axis, independently driven

**CAD Representation:**
- Full Gimbal render: outer rectangular frame + inner circular ring + payload sphere
- Show both rotation axes (Axis 1 = blue, Axis 2 = green)
- Animate each axis independently
- Show instantaneous gravity vector (red arrow) tumbling in 3D space
- Show time-averaged gravity vector shrinking toward zero (blue arrow fading)
- Color-coded frames: outer = dark steel, inner = matte aluminum, payload = amber/glass

**2D Physics — Three Operating Modes:**

Mode 1 — Constant Speed:
```
ω1 = constant, ω2 = constant (ω1 ≠ ω2, irrational ratio preferred)
g_vec(t) = R_z(ω1*t) * R_x(ω2*t) * [0, 0, -g]
g_residual = ||(1/T) ∫ g_vec(t) dt|| → 0 as T → ∞
```

Mode 2 — Random Walk:
```
At each interval Δt, randomly select new ω1, ω2 from range [-ω_max, +ω_max]
Direction changes are smooth (acceleration-limited)
Randomization ensures isotropic gravity distribution over time
```

Mode 3 — Optimized RPM (Johannes algorithm):
```
Minimize: ∫₀ᵀ ||g_avg(t)||² dt
Subject to: |ω| ≤ ω_max, |α| ≤ α_max
Uses dynamic programming or gradient descent on rotation sequence
```

**Residual gravity quality metric (display live):**
```
g_res% = (||g_avg|| / g_Earth) * 100
Target: g_res < 0.01g  (i.e., < 1% of Earth gravity)
```

## SECTION 4: PHYSICS ENGINE — EXACT FORMULAS

All of these must appear in MATHS_CALCULATIONS_README.md with citations.

### Quaternion Rotation (no gimbal lock)
```
q = [cos(θ/2), sin(θ/2)*nx, sin(θ/2)*ny, sin(θ/2)*nz]
q_total = q2 ⊗ q1  (Hamilton product)
v_rotated = q ⊗ v ⊗ q*
```
Source: Hamilton (1843), Kuipers (1999) "Quaternions and Rotation Sequences"

### Gravity Vector in Rotating Frame
```
1D: g(t) = [g*sin(ω*t), g*cos(ω*t), 0]
2D Constant: g(t) = R_outer(ω1*t) · R_inner(ω2*t) · [0, 0, -g]
2D Random: g(t) = R(θ1(t)) · R(θ2(t)) · [0, 0, -g]  where θ changes stochastically
```
Source: Briegleb (1992), Mesland (1992), Borst & van Loon (2009)

### Residual Gravity (Time-Averaged)
```
g_avg = (1/T) ∫₀ᵀ g(t) dt
g_res = ||g_avg||
Quality: g_res/g_Earth * 100 [%]
```
Source: Schwarzenberg-Czerny (1993), van Loon (2007)

### Centrifugal Acceleration
```
a_c = ω² · r
where r = distance from rotation center to sample geometric center
Requirement: r < 1mm for g_c < 0.001g at ω = 1 RPM
```
Source: Herranz et al. (2013), Wuest et al. (2017)

### Stokes Settling Velocity (biological sample sedimentation)
```
v_s = (2 * r_cell² * (ρ_cell - ρ_medium) * g) / (9 * μ)
Clinostat nullification condition: ω >> (2*g) / (π * v_s)
```
Source: Stokes (1851), Briegleb (1992) [CITE-BIO-001]

### Moment of Inertia (rotating frame)
```
I_ring = (1/2) * m * (R_outer² + R_inner²)
I_total = I_frame + I_payload + I_motor_rotor
α = τ / I_total
```
Source: Meriam & Kraige "Engineering Mechanics: Dynamics"

### PID Motor Control
```
e(t) = ω_setpoint - ω_actual
u(t) = Kp*e(t) + Ki*∫e(t)dt + Kd*(de/dt)
Anti-windup: clamp ∫e(t) to [-I_max, +I_max]
```
Source: Åström & Hägglund (1995) "PID Controllers: Theory, Design, and Tuning"

### Euler Angles to Rotation Matrix (for display only, not simulation)
```
R = Rz(ψ) · Ry(θ) · Rx(φ)
Gimbal lock occurs when θ = ±π/2 — NEVER use Euler for simulation, use quaternions
```
Source: Diebel (2006) "Representing Attitude: Euler Angles, Unit Quaternions..."

### Vibration / FFT Analysis
```
X[k] = Σ x[n] * e^(-j2πkn/N)   (DFT)
Use Welch's method for power spectral density
Resonance detection: flag peaks where S(f) > threshold
```
Source: Cooley & Tukey (1965), Welch (1967)

### Inertia Tensor (for 3D scaffold)
```
I = [[Ixx, -Ixy, -Ixz],
     [-Ixy, Iyy, -Iyz],
     [-Ixz, -Iyz, Izz]]
Ixx = ∫(y²+z²)dm, etc.
```
Source: Goldstein "Classical Mechanics", 3rd Ed.

## SECTION 5: MOTOR CONTROL & ETHERNET COMMUNICATION

### Hardware Assumed
- MCU: Raspberry Pi 4 / STM32H7 with Ethernet PHY
- Motor Driver: ODrive v3.6 or TMC5160 SPI
- Encoder: Incremental quadrature, 4096 CPR
- IMU: ICM-42688-P (6-axis, 32kHz output rate)
- Temperature: PT100 RTD on each motor
- Power: INA226 current/voltage sensor on each axis

### Communication Protocol (Ethernet/WiFi)
- Transport: TCP sockets (reliable) for commands, UDP for streaming telemetry
- Port: 8080 (commands), 8081 (telemetry stream)
- Packet format (JSON):
```json
{
  "packet_id": 1042,
  "timestamp_ms": 1234567890,
  "type": "CMD" | "ACK" | "TELEM" | "HEARTBEAT" | "ERROR",
  "payload": {
    "axis": 1,
    "rpm_setpoint": 5.0,
    "direction": "CW",
    "mode": "PID"
  },
  "crc32": "A3F2C1D0"
}
```
- CRC32 validation on every packet — reject and flag corrupted packets
- Heartbeat every 500ms — if missed 3 consecutive → CONNECTION LOST alert
- Command acknowledgement: every CMD must receive ACK within 100ms or retry (max 3)
- Telemetry stream: 10Hz minimum, 100Hz maximum (user selectable)
